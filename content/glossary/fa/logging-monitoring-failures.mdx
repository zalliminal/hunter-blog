## شکست‌های لاگ و مانیتورینگ چیه؟

خلاصه در یه جمله: وقتی ندونی چی تو سیستمت اتفاق می‌افته، هکر می‌تونه ماه‌ها اونجا باشه و تو نفهمی!

فرض کن یه خونه داری. هیچ دوربینی نداری، هیچ زنگی نداره، هیچکس هم چک نمی‌کنه کی وارد می‌شه. هکر می‌تونه بیاد، هر چی می‌خواد برداره، بره و تو هیچوقت نفهمی! لاگ و مانیتورینگ همینه — باید بدونی چی اتفاق می‌افته .

### انواع شکست‌های لاگ و مانیتورینگ:

**<span dir="ltr">۱</span>. لاگ نمی‌گیری:**
هیچ رویدادی ثبت نمی‌شه. اگه حمله‌ای بشه، نمی‌فهمی کی، چطور، از کجا!

**<span dir="ltr">۲</span>. لاگ‌های ناقص:**
فقط یه سری چیزا رو لاگ می‌کنی. مثلاً لاگین موفق رو می‌گیری، لاگین ناموفق رو نه. هکر <span dir="ltr">Brute</span> <span dir="ltr">Force</span> می‌کنه، تو نمی‌فهمی!

**<span dir="ltr">۳</span>. لاگ‌ها رو چک نمی‌کنی:**
لاگ می‌گیری ولی هیچکس نگاهشون نمی‌کنه. هکر <span dir="ltr">۶</span> ماهه تو سیستمت، تو خبر نداری!

**<span dir="ltr">۴</span>. <span dir="ltr">Alert</span> نداره:**
وقتی چیزی مشکوک اتفاق می‌افته، هیچ هشداری نمی‌گیری. باید خودت بری چک کنی!

**<span dir="ltr">۵</span>. لاگ‌ها دستکاری می‌شن:**
هکر وارد می‌شه، لاگ‌ها رو پاک می‌کنه تا ردش گم بشه!

**<span dir="ltr">۶</span>. اطلاعات حساس تو لاگ:**
پسورد، توکن، اطلاعات کارت بانکی رو لاگ می‌کنی. اگه لاگ لو بره، فاجعه‌ست!

### مثال واقعی:

فرض کن سایتت اینطوری لاگ می‌گیره:
- لاگین موفق رو لاگ می‌کنی
- لاگین ناموفق رو نه
- هیچ Alertی برای <span dir="ltr">۱۰۰۰</span> بار لاگین ناموفق نداری
- لاگ‌ها رو ماهی یه بار چک می‌کنی

هکر می‌تونه:
<span dir="ltr">۱</span>. <span dir="ltr">Brute</span> <span dir="ltr">Force</span> کنه (تو نمی‌فهمی)
<span dir="ltr">۲</span>. وارد بشه
<span dir="ltr">۳</span>. داده‌ها رو بدزده
<span dir="ltr">۴</span>. لاگ‌ها رو پاک کنه
<span dir="ltr">۵</span>. بره و تو هیچوقت نفهمی!

### چرا برای امنیت مهمه؟

چون شماره <span dir="ltr">۹</span> <span dir="ltr">OWASP</span> <span dir="ltr">Top</span> <span dir="ltr">10</span> <span dir="ltr">2021</span> هست! بدون لاگ و مانیتورینگ، تو کوری! نمی‌دونی کی حمله کرده، چطور حمله کرده، چی دزدیده. نمی‌تونی جواب بدی، نمی‌تونی جلوش رو بگیری، نمی‌تونی یاد بگیری برای بعد.
